# AXIS вҖ” Centralized Knowledge Layer

> One source of truth. Every team's context, in one place.

AXIS is a RAG-based (Retrieval-Augmented Generation) knowledge assistant that centralizes documentation from across your organization and makes it queryable through a chat interface. It syncs from **Jira, Confluence, Slack, Notion, and Google Drive**, retrieves the most relevant context for any question, and generates grounded, source-cited answers вҖ” using a **local LLM (Ollama) by default**, so it runs free with no API key.

---

## What it does

- **Five live knowledge sources** вҖ” Jira, Confluence, Slack, Notion, Google Drive (plus contributed docs)
- **Hybrid retrieval** вҖ” semantic vector search (ChromaDB) + BM25 keyword search, merged and re-ranked with a cross-encoder
- **Local-first answers** вҖ” generated by a local Ollama model (no API key, no cost); optionally Claude via the Anthropic API
- **Clickable citations** вҖ” every answer's source chips link back to the original Jira ticket / Confluence page / Slack message / Notion page / Google Doc
- **User accounts** вҖ” login/sign-up so each team member has their own session
- **Feedback loop** вҖ” рҹ‘Қ/рҹ‘Һ on answers; upvotes nudge useful docs higher in future retrieval
- **Background sync** вҖ” a scheduler re-syncs all sources every few hours
- **Clean React UI** + a FastAPI backend

## Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Choose an answer-generation backend

**Option A вҖ” Local Ollama (default, free, no API key):**

```bash
brew install ollama          # or download from https://ollama.com/download
ollama serve                 # start the local server (leave running)
ollama pull llama3.2         # ~2 GB; or llama3.1 (8B) for higher quality
```

`.env` (default values shown):
```
AXIS_LLM_BACKEND=ollama
OLLAMA_MODEL=llama3.2
OLLAMA_URL=http://localhost:11434
```

**Option B вҖ” Claude via Anthropic API:**
```
AXIS_LLM_BACKEND=anthropic
ANTHROPIC_API_KEY=sk-ant-...
```

> If the model is unreachable, AXIS degrades gracefully вҖ” it still returns the retrieved sources (with working citations and feedback) instead of erroring.

### 3. Ingest the seed docs (once)

```bash
python ingest.py
```

Embeds the `data/*.json` docs into a local ChromaDB store (`axis_db/`). First run downloads the embedding model (~30s).

### 4. Start the backend

```bash
set -a && . ./.env && set +a       # load .env into the shell
uvicorn api:app --reload --port 8000
```

### 5. Start the frontend

```bash
cd ui
npm install
npm run dev
```

Open **http://localhost:3000**, then **sign up** or log in.

## Connecting knowledge sources

All sync config lives in `.env`. Each source is optional вҖ” unconfigured ones are skipped gracefully. After configuring, click the source's button in the sidebar (or **Sync All**), or run `python sync.py`.

### Jira + Confluence
```
JIRA_BASE_URL=https://yourorg.atlassian.net
JIRA_EMAIL=you@yourorg.com
JIRA_API_TOKEN=...            # id.atlassian.com/manage-profile/security/api-tokens
JIRA_PROJECTS=ENG,DATA,PROD   # comma-separated project keys
CONFLUENCE_SPACES=ENG,DATA    # comma-separated space keys (same credentials)
```

### Slack
Create a Slack app with a **bot token** (`xoxb-вҖҰ`) and scopes `channels:history, channels:read, groups:history, users:read`. Invite the bot to each channel (`/invite @YourApp`).
```
SLACK_BOT_TOKEN=xoxb-...
SLACK_CHANNELS=eng-help=Engineering,C0123ABC=Data   # channel name OR ID = AXIS team
```

### Notion
Create an internal integration at notion.so/my-integrations, then **connect** each page to it (page вҖўвҖўвҖў вҶ’ Connections).
```
NOTION_TOKEN=ntn_...
NOTION_TEAM=Product           # AXIS team all Notion pages are filed under
```

### Google Drive
Create a GCP service account, enable the **Drive API**, download a JSON key, and **share** your Docs/folder with the service account's email.
```
GOOGLE_SERVICE_ACCOUNT_FILE=/path/to/service-account.json
GDRIVE_TEAM=Data
GDRIVE_FOLDER_ID=             # optional: limit to one folder
```

### Background auto-sync
```
AXIS_SYNC_INTERVAL_HOURS=6    # how often the scheduler re-syncs every source
```

### Auth
```
AXIS_JWT_SECRET=change-me-to-a-long-random-string   # signs login tokens
```

## How to use

- **Ask anything** вҖ” AXIS runs hybrid retrieval, re-ranks, and answers with source attribution.
- **Filter by team** in the sidebar to scope a question to one team's knowledge.
- **Click a source chip** (those with a вҶ—) to open the original Jira/Confluence/Slack/Notion/Drive item.
- **Rate answers** with рҹ‘Қ / рҹ‘Һ вҖ” upvoted docs get a small ranking boost on future questions; the effect is capped so it refines rather than overrides relevance.
- **Add Context** in the sidebar to contribute a doc directly.
- **Sync** individual sources or **Sync All** from the sidebar.

## Tech stack

| Component | Technology |
|---|---|
| Backend | FastAPI (`api.py`) |
| Frontend | React 19 + Vite (`ui/`) |
| Embeddings | `sentence-transformers` (all-MiniLM-L6-v2, local) |
| Re-ranking | cross-encoder (`ms-marco-MiniLM-L-6-v2`, local) |
| Keyword search | BM25 (`rank-bm25`) |
| Vector store | ChromaDB (persistent, local) |
| Answer LLM | Ollama (local, default) or Claude via Anthropic API |
| Auth / data | SQLite + PBKDF2 + JWT (PyJWT) |
| Language | Python 3.10+ |

> A legacy Streamlit UI (`app.py`) also exists from the original POC; the React UI + FastAPI backend is the current interface.

## Security notes

- Treat all tokens/keys as secrets. Keep `.env` and any service-account JSON **out of version control**.
- Set a strong `AXIS_JWT_SECRET` before any real use.
- Rotate provider credentials if they're ever exposed (Slack token, Notion token, Google service-account key).
